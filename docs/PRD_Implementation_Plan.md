## 一、市场调研与竞品分析

### 1.1 宠物营养市场规模

#### 全球宠物营养市场
- **2024年市场规模**：约1200亿美元
- **预计2030年**：将达到1800亿美元
- **年复合增长率**：约6.5%

#### 中国宠物营养市场
- **2024年市场规模**：约3000亿人民币
- **猫粮市场占比**：约35%（约1050亿人民币）
- **年增长率**：约15-20%（高于全球平均水平）

#### 市场趋势
1. **高端化趋势**：消费者愿意为高品质、功能性产品支付溢价
2. **个性化需求**：针对不同品种、年龄、健康状态的定制化产品需求增长
3. **健康意识提升**：预防性健康管理成为主流
4. **数字化服务**：AI驱动的个性化推荐服务需求旺盛

## 三、数据准确性与可购买性保障方案

### 3.1 核心原则

**一个人副业开发，必须做减法，聚焦最核心的问题**

1. **数据准确性是基础** - 没有准确数据，一切免谈
2. **可购买性是底线** - 推荐买不到的产品，用户会流失
3. **LLM幻觉防控是关键** - 这是AI产品的核心风险
4. **自动化为主，人工为辅** - 最小化人力消耗

### 3.2 产品数据准确性保障

#### 3.2.1 数据来源（一次性投入）

**单一可靠来源 + 人工验证**
- 只从天猫/京东官方旗舰店采集数据
- 每款产品人工核对一次成分表和价格
- 建立一个JSON文件管理产品数据

**为什么这样够用？**
- 官方旗舰店数据最可靠
- 30-50款产品，人工核对完全可行
- JSON文件简单直观，一个人就能维护

#### 3.2.2 数据验证（完全自动化）

**自动化验证规则**

1. **必填字段检查**
   - 产品名称、品牌、价格、成分表、营养成分
   - 缺一不可，否则不录入

2. **营养成分范围检查**
   - 蛋白质25-50%、脂肪8-25%、纤维1-10%
   - 超出范围标记，人工确认

3. **有害成分检查**
   - 检查是否含有BHA、BHT、乙氧基喹啉
   - 含有则不推荐

**实现方式：**
```typescript
// 数据验证函数
function validateCatFood(product: CatFood): ValidationResult {
  const errors: string[] = [];
  
  // 必填字段检查
  if (!product.name || !product.brand || !product.price) {
    errors.push('缺少必填字段');
  }
  
  // 营养成分范围检查
  if (product.protein < 25 || product.protein > 50) {
    errors.push('蛋白质含量超出正常范围');
  }
  
  // 有害成分检查
  if (hasHarmfulIngredients(product.ingredients)) {
    errors.push('含有有害成分');
  }
  
  return {
    valid: errors.length === 0,
    errors
  };
}
```

#### 3.2.3 数据更新（按需更新）

**更新触发条件：**
1. 用户反馈某款产品信息错误
2. 某款产品停产
3. 发现新的优质产品想加入

**平时完全不需要维护**

### 3.3 产品可购买性验证

#### 3.3.1 产品状态管理（按需更新）

**产品状态**
- 在售：正常推荐
- 停产：标记为"已停产"，不再推荐
- 缺货：标记为"暂时缺货"，但仍可推荐

**更新触发条件：**
- 用户反馈"买不到" → 标记为"缺货"
- 用户反馈"已停产" → 标记为"停产"
- 发现链接失效 → 标记为"链接失效"

#### 3.3.2 购买链接（一次性设置）

**初始设置**
- 一次性录入所有产品的购买链接
- 链接来自官方旗舰店，相对稳定

**更新触发条件：**
- 用户反馈"链接打不开" → 更新链接
- 发现链接失效 → 更新链接

#### 3.3.3 价格（可选，不影响推荐）

**价格的作用**
- 只是给用户参考
- 不影响推荐结果
- 不更新也没关系

**更新策略**
- 完全不更新，或者
- 用户反馈价格不对时才更新

### 3.4 LLM推荐准确性验证

#### 3.4.1 候选集约束（必须做）

**严格限制LLM只能从数据库推荐**
- LLM提示词明确说明：只能从提供的候选集中选择
- 禁止LLM自由生成产品名称
- 候选集就是你的JSON文件中的产品

**这是防止幻觉的第一道防线，必须做**

**实现方式：**
```typescript
// 候选集约束
const candidateProducts = catFoodDatabase.filter(product => {
  return product.lifeStage.includes(cat.ageStage) &&
         product.functionalTags.some(tag => cat.healthTags.includes(tag));
});

// LLM提示词
const prompt = `
你只能从以下候选集中选择产品：
${candidateProducts.map(p => p.name).join(', ')}

禁止：
- 禁止生成不在候选集中的产品名称
- 禁止编造产品信息
`;
```

#### 3.4.2 推荐理由验证（完全自动化）

**关键词匹配验证**
- 提取推荐理由中的关键词（蛋白质、脂肪、成分等）
- 与产品实际数据对比
- 如果差异超过2%，标记为"可能不准确"

**实现方式：**
```typescript
// 推荐理由验证
function validateRecommendationReason(
  reason: string,
  product: CatFood
): ValidationResult {
  const errors: string[] = [];
  
  // 提取推荐理由中的蛋白质含量
  const proteinMatch = reason.match(/蛋白质(\d+)%/);
  if (proteinMatch) {
    const reasonProtein = parseInt(proteinMatch[1]);
    const diff = Math.abs(reasonProtein - product.protein);
    if (diff > 2) {
      errors.push(`蛋白质含量差异过大：推荐${reasonProtein}%，实际${product.protein}%`);
    }
  }
  
  return {
    valid: errors.length === 0,
    errors
  };
}
```

#### 3.4.3 营养匹配验证（完全自动化）

**基础营养需求检查**
- 幼猫：蛋白质≥30%
- 成猫：蛋白质≥26%
- 老年猫：蛋白质≥26%，易消化
- 减重：脂肪≤12%
- 泌尿道健康：镁≤0.1%

**如果推荐产品不满足基础需求，标记为"不推荐"**

**实现方式：**
```typescript
// 营养需求检查
function checkNutritionalRequirements(
  product: CatFood,
  cat: Cat
): ValidationResult {
  const errors: string[] = [];
  
  // 幼猫蛋白质需求
  if (cat.ageStage === 'kitten' && product.protein < 30) {
    errors.push('幼猫需要蛋白质≥30%');
  }
  
  // 减重脂肪需求
  if (cat.healthTags.includes('weight_loss') && product.fat > 12) {
    errors.push('减重需要脂肪≤12%');
  }
  
  // 泌尿道健康镁含量
  if (cat.healthTags.includes('urinary_health') && product.magnesium > 0.1) {
    errors.push('泌尿道健康需要镁≤0.1%');
  }
  
  return {
    valid: errors.length === 0,
    errors
  };
}
```

#### 3.4.4 人工抽检（最有效）

**每天抽检5-10个推荐结果**
- 检查推荐产品是否在数据库中
- 检查推荐理由是否与产品数据一致
- 检查推荐是否合理

**为什么这样够用？**
- 一个人每天花10-15分钟就能完成
- 人工抽检比任何自动化验证都可靠
- 发现问题及时调整提示词

### 3.5 准确性评分

#### 3.5.1 三级评分

**优秀（可直接展示）**
- 推荐产品在数据库中
- 推荐理由与产品数据一致
- 满足猫咪基础营养需求

**良好（可展示，标注提示）**
- 推荐产品在数据库中
- 推荐理由与产品数据基本一致（差异<5%）
- 满足猫咪基础营养需求

**不合格（不展示，重新生成）**
- 推荐产品不在数据库中
- 推荐理由与产品数据严重不符
- 不满足猫咪基础营养需求

#### 3.5.2 用户反馈（最简单的方式）

**推荐结果页加一个"这个推荐有用吗？"按钮**
- 有用：记录为正面反馈
- 没用：记录为负面反馈，人工查看原因

**每周统计一次**
- 正面反馈率>70%：继续优化
- 正面反馈率<50%：需要调整推荐策略

#### 3.6.3 真实场景

**最理想情况（用户反馈好）**
- 每周花5分钟看看反馈统计
- 几乎不需要处理任何问题
- **人力消耗：5分钟/周**

**一般情况（偶尔有问题）**
- 每周花10分钟看看反馈统计
- 每月花20分钟处理1-2个问题
- **人力消耗：30分钟/月**

**最差情况（问题较多）**
- 每周花10分钟看看反馈统计
- 每月花1小时处理5-10个问题
- **人力消耗：1.5小时/月**

### 3.7 关键成功因素

#### 3.7.1 数据质量是生命线
- 宁可少推荐几款，也要保证数据准确
- 每款产品都要人工核对
- 发现错误立即修正

#### 3.7.2 简单的验证比复杂的验证更可靠
- 候选集约束是最有效的防幻觉手段
- 人工抽检比自动化验证更可靠
- 用户反馈是最好的验证

#### 3.7.3 定期维护比实时监控更重要
- 每月花2-3小时维护数据
- 每天花10-15分钟抽检推荐
- 每周花30分钟分析反馈

#### 3.7.4 逐步迭代，不要追求完美

## 四、PRD修改建议

### 4.1 产品定位调整

**原定位：**
> "一个记住你家猫全部健康状态的 AI 营养师，在每个生命阶段推荐最适合的猫粮和罐头。"

**建议修改为：**
> "基于你家猫咪健康状态的AI营养顾问，提供个性化猫粮推荐和长期健康管理。"

**修改理由：**
- 强调"健康状态"而非"全部健康状态"（降低用户期望）
- 突出"长期健康管理"（差异化优势）
- 更符合MVP功能范围

### 4.2 新增章节：数据准确性与可购买性保障

**建议在PRD中新增以下章节：**

#### 4.2.1 第十七章：数据准确性保障

**17.1 数据来源**

- 人工核对：每款产品人工核对一次成分表和价格

#### 4.2.2 第十八章：可购买性验证

**20.3 反馈统计**

## 六、总结

### 6.1 核心要点

1. **市场机会明确**：宠物营养市场规模大，增长快，个性化推荐需求旺盛
2. **差异化优势明显**：AI驱动的深度推荐、健康上下文、中立性
3. **技术方案可行**：现有技术栈完全支持，一个人副业开发可行
4. **数据准确性有保障**：候选集约束 + 自动化验证 + 人工抽检
5. **可购买性有保障**：产品状态管理 + 购买链接验证 + 用户反馈
6. **人力消耗可控**：初始9-14小时，日常15-60分钟/月

### 6.2 关键成功因素

1. **数据质量是生命线**：宁可少推荐几款，也要保证数据准确
2. **简单的验证比复杂的验证更可靠**：候选集约束是最有效的防幻觉手段
3. **定期维护比实时监控更重要**：按需更新，不定期维护
4. **逐步迭代，不要追求完美**：先保证核心功能可用，再逐步优化

### 6.3 风险控制

1. **LLM幻觉风险**：候选集约束 + 推荐验证 + 人工抽检
2. **数据准确性风险**：人工核对 + 自动化验证 + 用户反馈
3. **产品可购买性风险**：产品状态管理 + 购买链接验证 + 用户反馈
4. **时间投入风险**：固定维护时间，忙的时候暂停更新

### 6.4 下一步行动

1. **立即开始**：按照实施路线图，第1周开始数据准备
2. **持续优化**：根据用户反馈，持续优化推荐算法
3. **逐步扩展**：MVP验证成功后，逐步扩展功能
4. **商业化探索**：v1.5开始探索付费模式
